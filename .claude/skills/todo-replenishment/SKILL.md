# TODO の補充

## トリガー

`TODO` が空の状態でユーザーから作業依頼を受けたとき。

## アクション

1. 現在の実装を分析する (変換対応状況、テストカバレッジ、既知の制約、技術的負債)
2. プロジェクトの目的 (Ruby → TypeScript トランスパイラ) を考慮し、次に価値のある機能・改善を提案する
   - 未対応の Ruby 構文
   - 品質改善 (テスト、エラーハンドリング、変換精度)
   - 開発基盤の整備
3. ユーザーにヒヤリングし、優先度と方向性を確認する
4. 合意した項目を `TODO` に書き込む

## 禁止事項

- ユーザーへのヒヤリングをせずに `TODO` に項目を書き込むこと
- 分析結果を提示せずに「何をやりましょうか？」とだけ聞くこと
