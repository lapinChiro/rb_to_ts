# テスト規約

## トリガー

テストコードを新規作成または変更するとき。

## アクション

以下の規約に従ってテストを配置・記述する:

### 配置

| 種類 | 配置 | 用途 |
|------|------|------|
| **ユニットテスト** | `src/**/*.test.ts` | モジュール内部ロジック |
| **統合テスト** | `tests/**/*.test.ts` | パイプライン全体の統合テスト |
| **スナップショットテスト** | `tests/` (vitest) | Ruby → TS 変換の出力検証 |
| **E2E テスト** | `tests/e2e/` | 変換後 TS の実行時正確性検証 |

### スナップショットテスト

- fixture ファイル: `tests/fixtures/<name>.input.rb`
- `toMatchSnapshot()` で出力を検証
- スナップショット更新: `npx vitest run -u`

### E2E テスト

E2E テストは「同じロジックを Ruby で実行した stdout」と「変換後の TS を `npx tsx` で実行した stdout」が一致することを検証する。

- 変換機能を変更したら、対応する E2E テストの追加・拡充が必須

### コード規約

- 各テストは独立して実行可能であること。テスト間で状態を共有しない
- テスト名: `describe('<対象>') > it('<条件>のとき<期待結果>')`

## 禁止事項

- テスト間で可変な状態 (ファイル、グローバル変数等) を共有すること
- 変換機能の変更で E2E テストを書かずに完了とすること
