# ドキュメントの原則

## 適用条件

ソースコードを修正・変更・追加・削除したとき。

## 制約

ドキュメントに少しでも誤った情報があると、ドキュメント全体の信頼性が損なわれる。

1. 変更の影響範囲をドキュメントを含めて確認し、影響を受ける全てのドキュメントを更新する
2. ドキュメント更新はコード変更と同じタイミングで行う

確認対象:
- CLAUDE.md (アーキテクチャ、コマンド、規約)
- README.md
- plan.md
- コードのドキュメントコメント (JSDoc 等)

## 禁止事項

- ドキュメント更新を後回しにすること
- コード変更の影響範囲にあるドキュメントを確認せずに作業完了とすること
