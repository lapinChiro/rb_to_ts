# デバッグ・エラー修正の原則

## 適用条件

修正が 1 回で成功しなかったとき。

## 制約

次の修正を試みる前に立ち止まり、以下の順序で進めること:

1. **情報収集**: バージョン、依存関係、環境情報など、関連する事実を集める
2. **仮説構築**: 集めた情報から根本原因の仮説を立てる
3. **仮説検証**: 仮説を検証するコマンドを実行する
4. **修正**: 根本原因が特定できてから修正する

## 禁止事項

- アドホックに「エラーメッセージに書いてあることを直す」を繰り返すこと
- 表面的な症状だけを見て修正すること (なぜそのエラーが起きているかを考えよ)
- 同じ修正を 2 回繰り返すこと
- 根本原因の仮説を立てずに次の修正を試みること
